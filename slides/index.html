24-hour hackathon build

ForemanAI

AI for the trades. Human in the loop.

The problem

Most field service shops still run on paper and memory.

2-4 days
between finishing a job and sending the invoice
~30%
of invoices come back with questions before they get paid

A tech wraps up at 5pm. The invoice gets typed up that night, if they remember, missing half the parts, wrong rate, nothing consistent with last month's.

It starts with a text

No form. No app. Just a text.

incoming · SMS · 09:14
Repair the HVAC unit at 142 Elm Street. The compressor is making a grinding noise and the system stopped cooling yesterday. Client: Riverside Property Management. Urgency: high.

Foreman takes that and turns it into a complete, ready-to-send invoice. A human approves each step before anything moves.

The pipeline

Three agents, one shared work order in Redis. They don't call each other; they read and write the same object.

Each agent writes its section and stops. The next one picks up from there.

The shared work-order object

One object in Redis. Each agent owns one section and doesn't touch the others.

uuid
status

raw_request

Step 1 — Intake

Reads the text, pulls out what it knows, flags what's missing. Nothing moves until a human says go.

{
  "job_type": "hvac_repair",
  "client": "Riverside Property Management",
  "location": "142 Elm Street",
  "urgency": "high",
  "completeness": {
    "has_contact": true,
    "has_location": true,
    "has_description": true,
    "missing": []
  },
  "status": "awaiting_approval"
}

Structured and staged. Waiting on a human.

Step 2 — Scheduling

Picks time slots, writes the client message, estimates what parts you'll probably need.

PARTS SUGGESTION
compressor unit     $420–$680
capacitor           $15–$45
refrigerant R-410A  $60–$120
                    ─────────
estimated range     $495–$845
OUTREACH DRAFT
Hi Riverside PM — we've received your request for 142 Elm Street. A tech can be on-site Thu 9am–11am or Fri 2pm–4pm. Please confirm.

Still nothing sent. Human approves first.

Step 3 — Invoicing

It already knows the job, the client, the parts. It just asks for what's actually missing.

GAP-FILL CHAT
agent → I've pre-filled parts and job description from the work order. What's your labor rate and hours on site?
tech → $140/hr, 3.5 hours, $75 trip charge
CONSISTENCY CHECK
⚠ LABOR_RATE_HIGH
$140/hr is above the historical average of $98/hr for HVAC repair in this region.
Doesn't stop you. Just flags it. Your call.

Step 4 — ArmorIQ block

Before the agent commits anything, ArmorIQ checks it against what was approved. If it's not in the plan, it stops.

armoriq → checking action: FILL_TEMPLATE
armoriq → plan hash: 7f3a…c9b1 ✓
armoriq → checking action: DEMO_BLOCK
✗ BLOCKED — action not in approved plan
operator alert raised · agent cannot proceed · waiting for human

This isn't a system prompt asking it nicely to behave. The action physically can't go through without clearance.

Step 5 — Human approves

One button. Invoice comes out clean, email is drafted and waiting. Nothing left your hands without you saying so.

Every decision is traced
